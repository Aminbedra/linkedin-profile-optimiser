# Keyword and Recruiter Discoverability

Goal: make the profile surface when the right people search. Treat keywords like seeds. They only grow where you plant them, so placement matters as much as choice.

## How search works (in plain terms)

LinkedIn search and the separate Recruiter product both match queries against the words on a profile, weighting some fields more heavily. The exact algorithm is private and changes, so treat the below as well-established best practice, not guaranteed ranking rules. State this honestly to the user.

High-signal fields, roughly in order:
1. Headline
2. Job titles in the Experience section
3. About section
4. Skills
5. Location and current/past company

## Method

1. **Derive the keyword set from the target role.** Take the user's target job title or client need and list the words a recruiter or buyer would actually type. Pull from real job adverts for that role where possible.
2. **Audit current placement.** For each keyword, check whether it appears in the high-signal fields above. Build a simple presence table: keyword, where it appears now, where it should appear.
3. **Recommend specific placements.** Do not say "use more keywords." Say exactly which keyword goes into which field, and supply the rewritten line.
4. **Avoid stuffing.** Keywords must read naturally. A profile written for robots repels humans, and humans make the hiring decision. Aim for natural repetition across fields, not a keyword list.

## Output

Present a short table of target keywords and their recommended placement, then the rewritten lines that achieve it. Close with a one-line caveat that rankings cannot be guaranteed.
